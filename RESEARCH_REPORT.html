🔴 Bug #1 — JSC DataView Integer Overflow (Goydish)

JSC Integer Overflow → OOB Write · CVE Pending
TIER 1 — CRÍTICO
📧 Fonte: seclists.org/oss-sec/2025/q4/292 — Joseph Goydish II — 26 Dez 2025
📁 Arquivo: Source/JavaScriptCore/runtime/JSArrayBufferView.cpp / JSDataView.cpp
oss-secNo-JIT ✓PS4 SEM GIGACAGE
Integer overflow em index * elementSize no DataView e TypedArray. O valor 0xFFFFFFFE + 4 resulta em overflow de 32 bits, passando bounds checks. Em iOS/macOS, o Gigacage (namespace 31) age como guardião — detecta o acesso além de 16GB e gera EXC_GUARD terminando o processo. No PS4, o Gigacage não está implementado — o overflow passa direto para o hardware sem proteção, criando potencial primitivo de escrita arbitrária.
// CÓDIGO VULNERÁVEL — JSArrayBufferView.cpp // Falta de checked arithmetic no cálculo do offset: size_t byteOffset = index * m_elementSize; // OVERFLOW aqui if (byteOffset >= m_byteLength) return error; // check passa com valor truncado m_vector[byteOffset] = value; // write fora dos limites
// FIX PROPOSTO (Goydish): size_t byteOffset; if (__builtin_mul_overflow(static_cast(index), m_elementSize, &byteOffset)) return throwOverflowError(); // PoC ORIGINAL (3 linhas): const buffer = new ArrayBuffer(1024); const view = new DataView(buffer); view.setUint32(0xFFFFFFFE, 0x41414141); // trigger

🔴 Bug #2 — Array.toReversed() Uninit Memory (qriousec)

Uninitialized Butterfly Slots → Heap Leak + RCE
TIER 1 — CRÍTICO
📝 Fonte: qriousec.github.io/post/jsc-uninit — Jan/2024
🔧 Bug commit: 053d9a84 · Fix commit: 9158c52 · Arquivo: Source/JavaScriptCore/runtime/JSArray.cpp
Blog TécnicoNo-JIT ✓
fastToReversed() aloca um butterfly com vectorLength = max(3, publicLength), mas só inicializa os primeiros publicLength slots via memcpy. Para um array de comprimento 1, slots [1] e [2] ficam com valores não-inicializados da heap (tipicamente 0xbadbeef0 ou ponteiros de alocações anteriores). Acessar v2[2] causa crash ou vaza ponteiros de heap — primitivo de leak sem JIT confirmado pelo qriousec.
// BUG em fastToReversed() — JSArray.cpp: auto vectorLength = Butterfly::optimalContiguousVectorLength(resultStructure, length); // vectorLength = 3 quando length = 1 (BASE_CONTIGUOUS_VECTOR_LEN = 3) butterfly->setVectorLength(vectorLength); // aloca 3 slots butterfly->setPublicLength(length); // expõe 1 slot publicamente memcpy(resultData, srcData, sizeof(JSValue) * length); // copia apenas 1 slot // slots [1] e [2] ficam UNINIT — 0xbadbeef0 ou ponteiro de heap
// FIX (commit 9158c52): memcpy(resultData, srcData, sizeof(JSValue) * length); // ADICIONADO: zera slots extras após memcpy for (unsigned i = length; i < vectorLength; ++i) resultData[i] = JSValue(); // inicializa com undefined/zero // PoC: const v1 = [-1n]; const v2 = v1.toReversed(); v2[2].print(); // CRASH: 0xbadbeef0 desreferenciado como objeto

🟠 Bug #3 — CSS Anchor Popover nullptr (Bugzilla #279588)

renderer() nullptr → WebProcess Crash
TIER 2 — ALTO
🐛 Fonte: bugs.webkit.org/show_bug.cgi?id=279588 — Pawel Lampe — Set/2024
🔧 Fix: commit ba8e7f03 (285334@main) — Out/2024 · SEM CVE FORMAL
Bugzilla #279588No-JIT ✓ANTERIOR AO FIX
Ao esconder um popover que usa CSS anchor positioning, o StyleTreeResolver tenta processar o elemento para recalcular posicionamento de âncora, mas element.renderer() retorna nullptr porque o elemento já está em processo de destruição. Null dereference → WebProcess crash. Fix em Out/2024 com commit ba8e7f03 — o branch PS4 (Set/2023) é anterior ao fix.
// CRASH EM: Source/WebCore/style/StyleTreeResolver.cpp:1340 // element.renderer() retorna nullptr quando popover é escondido auto* renderer = element.renderer(); renderer->updateStyle(...); // NULL DEREFERENCE aqui!
// FIX (ba8e7f03) em AnchorPositionEvaluator.cpp: // "When anchor-positioned element is hidden using display:none, // there's no point processing it in anchoring anymore." if (!element.renderer()) return; // skip se sem renderer

🟠 Bug #4 — CSS Nesting Hoisting (Safari 18.2)

Style Resolver Exponential Complexity → OOM/Crash
TIER 2 — ALTO
🔧 Fonte: Safari 18.2 Release Notes · WebKit commit 300297@main · Sem CVE
Release NotesNo-JIT ✓
Safari 17 usa "hoisting behavior" no CSS Nesting: seletores filhos são embrulhados em :is() automaticamente. Isso pode causar explosão combinatorial de regras quando combinado com :has(), :not() e nesting profundo. Safari 18.2 removeu este comportamento no commit 300297@main. O PS4 branch (Safari 17) ainda usa o comportamento vulnerável.

🟠 Bug #5 — Temporal API Edge Cases

JSC Temporal — Implementação Parcial Safari 17 → Crashes
TIER 2 — ALTO
📝 Fonte: Igalia WebKit Periódico · Commits 2023-2024 sem CVE formal
Igalia BlogNo-JIT ✓
Temporal foi implementado parcialmente no Safari 17 com spec em flux. Edge cases em Duration.add() com relativeTo e valores extremos, Instant.fromEpochNanoseconds() com BigInt além do limite, e ZonedDateTime com transições DST ambíguas podem causar exceções não tratadas, assertion failures, ou crashes no JSC runtime.

🟡 Bugs #6 e #7 — Pesquisa Original

Intl.Segmenter OOM + RegExp/Proxy/WeakRef Chain
TIER 3 — EXPLORER
Pesquisa OriginalNo-JIT ✓
Bug #6: Intl.Segmenter com grapheme granularity e scripts Unicode complexos (Emoji ZWJ chains, Tibetano+Arabic mix, variation selectors) causa alocações O(n²) e possível OOM no PS4 com memória limitada.

Bug #7: Combinação de RegExp named lookbehind + Proxy interceptando .groups + WeakRef + FinalizationRegistry + GC forçado durante callbacks. Cria race condition entre GC e acesso a match objects — técnica não testada no PS4 e não referenciada em nenhum writeup de jailbreak.

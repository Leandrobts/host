🔧 CVE-2023-28205 — Patch Real (Commit c9880de)

CVE-2023-28205 — UAF no Structured Clone Deserializer
🔗 Commit: github.com/WebKit/WebKit/commit/c9880de4a28b9a64a5e1d0513dc245d61a2e6ddb
📁 Source/WebCore/bindings/js/SerializedScriptValue.cpp
📁 Source/JavaScriptCore/runtime/MarkedVector.h (novo arquivo)
Patch Público No-JIT Explorado ITW
O deserializador do structured clone armazenava ponteiros para objetos JS em Vectors simples não-escaneados pelo GC. Se o GC fosse disparado durante a deserialização, os objetos eram coletados enquanto o deserializador ainda tinha ponteiros para eles — UAF clássico. Descoberto por Clément Lecigne (Google TAG) + Donncha Ó Cearbhaill (Amnesty International).
❌ CÓDIGO VULNERÁVEL (antes do patch)
// CloneDeserializer::deserialize() em SerializedScriptValue.cpp // Estes Vectors NÃO são escaneados pelo GC: Vector<JSObject*, 32> outputObjectStack; // JSObject* sem GC root Vector<JSMap*, 4> mapStack; // JSMap* sem GC root Vector<JSSet*, 4> setStack; // JSSet* sem GC root // Se GC ocorrer aqui → objetos liberados → ponteiros dangling // Desserializador continua usando os ponteiros → USE-AFTER-FREE
✅ CÓDIGO CORRIGIDO (após c9880de)
// Substituídos por MarkedVector — GC-aware, registra objetos como GC roots: MarkedVector<JSObject*> outputObjectStack; // GC sabe que estes objetos estão em uso MarkedVector<JSMap*> mapStack; // GC não pode coletar entradas aqui MarkedVector<JSSet*> setStack; // Idem para Sets // MarkedVector é basicamente um Barrier de escrita que registra // o Vector no GC Heap, impedindo coleta prematura dos objetos
⚠️ ANÁLISE PS4
Patch aplicado no Safari 16.4.1 (Abril 2023). O PS4 13.04 usa Version/17.0 (Safari 17, Set/2023) — o fix PROVAVELMENTE chegou no branch. Porém: o seregonwar confirma que Safari 15.4 era vulnerável. Se o PS4 usa um branch congelado que não incluiu c9880de, o bug está presente. O PoC CVE-2023-28205.html detecta isso: crash via GC+postMessage confirma vulnerabilidade.

🔧 CVE-2025-24201 — Mecanismo ANGLE/WebGL

CVE-2025-24201 — OOB Write em ANGLE (WebGL PRIMITIVE_RESTART)
🔗 PoC: github.com/The-Maxu/CVE-2025-24201-WebKit-Vulnerability-Detector-PoC
📁 Source/WebCore/platform/graphics/angle/GraphicsContextGLANGLE.cpp
Patch Público No-JIT
❌ VALIDAÇÃO AUSENTE (vulnerável)
// WebGLRenderingContext::enable() em WebGL1 não valida: void WebGL1RenderingContext::enable(GCGLenum cap) { // FALTAVA: verificar se cap é válido para WebGL1 // 0x8D69 (PRIMITIVE_RESTART_FIXED_INDEX) é WebGL2 only // mas era aceito sem GL_INVALID_OPERATION m_context->enable(cap); // passava direto para o driver ANGLE }
✅ DETECÇÃO (3 linhas — The-Maxu)
var gl = canvas.getContext("webgl"); // contexto WebGL1 gl.enable(0x8D69); // PRIMITIVE_RESTART_FIXED_INDEX var err = gl.getError(); // 0 = VULNERÁVEL, 0x0500 = protegido // Se err === 0: implementação aceita enum inválido → OOB write possível
Relevância PS4: WebGL ausente no FW 13.04 — impacto direto mínimo. Porém o padrão (enum inválido aceito sem validação na camada gráfica) pode existir em outros componentes do GNM (GPU Native Mode) do PS4.

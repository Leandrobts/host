🟠 Bug #4 — CSS Nesting Hoisting: Style Resolver Stress

Fonte: Safari 18.2 Release Notes · WebKit commit 300297@main · Sem CVE formal

🔧 CONTEXTO TÉCNICO — Safari 17 branch vs 18.2 Safari 18.2 removeu o "hoisting behavior" do CSS Nesting onde seletores complexos
eram embrulhados automaticamente em :is() antes de serem processados.
No Safari 17 (PS4 branch), este comportamento AINDA EXISTE:
.parent { .child { color: red } } → hoisted para .parent:is(.child) { color: red }

ABUSO: seletores :has() + :is() + nesting profundo → explode combinações no resolver
Complexidade exponencial em number of rule checks → OOM ou crash no style resolver

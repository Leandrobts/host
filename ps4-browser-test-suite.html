Informações do Sistema
Dados coletados automaticamente do navegador. Compare os valores entre o Chrome desktop e o PS4 — diferenças indicam que o servidor pode tratar os dois de forma diferente.
Crash / DoS
Testa comportamentos que podem causar travamento, congelamento ou fechamento forçado do browser. No PS4, o WebKit antigo é mais suscetível a edge cases de parsing.
CSS — Valores extremos
crash
Injeta elemento com dimensões absurdas. Versões antigas do WebKit podem travar ao tentar calcular o layout.
DOM — Aninhamento profundo
crash
Cria 5000 elementos aninhados recursivamente. Pode estourar a pilha de recursão do motor de renderização.
SVG — Recursão via <use>
crash
Referência circular em SVG com <use href="#self">. Pode causar loop infinito no parser de SVG do WebKit.
Table — Aninhamento excessivo
crash
Tabelas HTML com 200 níveis de aninhamento. O algoritmo de layout de tabelas do WebKit pode exceder limites internos.
String — Atributo gigante
crash
Cria elemento com atributo de 500.000 caracteres. Testa o limite de tamanho de strings do DOM.
Canvas — Dimensões extremas
crash
Canvas com width/height máximo (32767px). Testa o limite de alocação de memória para buffers gráficos.
Media Parsing
Testa o comportamento do browser ao lidar com mídia malformada, formatos não suportados e URLs de mídia incomuns.
Video — Src malformado
media
Elemento <video> com src vazio, nulo e com caracteres especiais. Registra eventos de erro e como o browser os trata.
Audio — Múltiplos formatos
media
Verifica quais codecs de áudio o browser suporta via canPlayType(). Revela informações da versão do WebKit.
Video — Múltiplos formatos
media
Mesma verificação para vídeo: H.264, VP8, VP9, AV1, HEVC. Diferenças entre PS4 e Chrome desktop indicam código de servidor diferente.
Image — Formatos incomuns
media
Tenta carregar imagens em WebP, AVIF e APNG. Registra quais formatos são suportados e como o browser trata os não suportados.
JavaScript Engine
Testa limites, APIs ausentes e comportamentos específicos do motor JavaScript do PS4. O motor é baseado no JavaScriptCore (JSC) em versão antiga.
APIs — Suporte geral
JS
Verifica presença de APIs modernas: Fetch, Promise, WebSocket, WebRTC, ServiceWorker, etc. APIs ausentes revelam a versão do motor.
TypedArray — Limites de memória
JS
Tenta alocar ArrayBuffers de tamanhos crescentes até encontrar o limite. Revela a memória disponível para JS no console.
Regex — ReDoS timing
JS
Testa regex com backtracking catastrófico. Mede o tempo de execução — diferenças grandes indicam versões do motor com comportamento distinto.
Error — Stack trace format
info
Captura o formato de stack trace de erros. O formato varia entre versões do JSC e pode revelar informações sobre a versão interna do engine.
Date — Timezone e locale
info
Coleta timezone configurado no console, locale do sistema e formato de datas. Pode ser usado para fingerprinting do dispositivo.
WebDatabase — openDatabase
JS
O PS4 suporta a Web SQL Database API (removida do Chrome). Testa se a API existe e qual o limite de storage permitido.
CSS Rendering
Testa propriedades CSS modernas e edge cases de renderização. O WebKit antigo do PS4 não suporta várias propriedades introduzidas após 2016.
CSS Grid / Flexbox
CSS
Verifica suporte a CSS Grid, Flexbox gap, subgrid. Diferenças revelam a versão do motor de layout.
CSS Filter / Transform
CSS
Testa filter com valores inválidos e transform com matrizes contendo NaN. Comportamentos inesperados indicam bugs no compositor.
CSS Variables (custom props)
CSS
CSS Custom Properties foram adicionadas ao WebKit em 2016. Se não suportadas, revela firmware bem antigo — potencial para mais bugs.
CSS — Seletor gigante
crash
Injeta uma regra CSS com seletor de 10.000 classes encadeadas. Pode causar lentidão extrema ou crash no parser de CSS.
Storage / Limits
Descobre os limites exatos de armazenamento do browser do PS4. Limites diferentes do padrão podem indicar implementação customizada.
localStorage — Limite de tamanho
info
Descobre o limite exato do localStorage no PS4 por busca binária. O padrão é 5MB mas implementações customizadas variam.
Cookie — Limites e flags
info
Testa quantos cookies podem ser criados, tamanho máximo por cookie e comportamento de flags SameSite/Secure.
IndexedDB — Suporte e limites
info
Verifica se IndexedDB é suportado e funcional. Algumas versões do PS4 WebKit têm suporte parcial com bugs conhecidos.
sessionStorage — Persistência
info
Testa comportamento do sessionStorage — tamanho máximo e se persiste entre navegações (comportamento diferente do esperado no PS4).
Network Behavior
Testa como o browser do PS4 lida com requisições HTTP, CORS, redirecionamentos e protocolos.
Fetch — Suporte e headers
info
Verifica se Fetch API existe, quais headers são enviados por padrão e como o browser lida com respostas CORS.
WebSocket — Suporte
info
Testa presença e versão do WebSocket. Implementações antigas têm comportamentos diferentes em relação a reconexão e erro.
Performance — Timing API
info
Mede precisão do performance.now() e navigation timing. Diferenças de precisão podem ser usadas para side-channel attacks.

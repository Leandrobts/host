02 — GC Timing Map — Quando e Como o GC Coleta

🔍 O QUE OBSERVAR:
• alive: X/Y após cada gc() → previsibilidade do coletor
• cleanup callbacks → se aparecem significa que o GC rodou de verdade
• alive constante entre execuções → GC determinístico = exploitável
• timing (ms) → quanto tempo demora uma pressão de 512MB
• crash durante gc() → pressão de memória causa instabilidade
• Δ alive entre checkpoints → quantos objetos o GC coleta por ciclo

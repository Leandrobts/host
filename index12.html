PS4 WebKit v8 — Testes Diagnósticos de Alta Informação

FW 12.00 · WebKit 605.15 · LLInt · Sem JIT · Sem SAB

O QUE OBSERVAR — GUIA RÁPIDO

🔴 CRASH (P0) — Navegador fecha. Anotar: qual arquivo, qual botão (T01–T15), última linha visível no log.
🟡 EXCEÇÃO AUSENTE (P1) — Operação inválida não gerou exceção, retornou valor. Anotar: o valor retornado em hex + contexto. Ex: OOB read idx=14 → 0x1a3b00004f20 (esse é um ponteiro de heap!)
🟡 VALOR HEX DE HEAP (P1) — Qualquer valor no range 0x1000–0x1_0000_0000_0000 que aparecer em leitura OOB é um endereço de memória real. Anotar o valor exato e o índice.
🔵 COMPORTAMENTO INESPERADO (P2) — Resultado diverge da spec JavaScript/DOM. Anotar o log completo do teste.
🔵 GC TIMING (P2) — Nos testes de WeakRef/FinalizationRegistry, anotar os números alive: X/Y em cada checkpoint. Previsibilidade = exploitabilidade.
⚪ TIMINGS (P3) — Tempos de operação. Anotar quando uma operação demora 2× ou mais que o esperado (sinaliza fallback de código ou bailout de otimização).

INTERPRETANDO VALORES HEX

  • 0x0001_0000_0000_0000 a 0x0007_FFFF_FFFF_FFFF → ponteiro de heap JSC válido
  • 0x0108_2407_0000_0000 → JSCell header de array (JIT layout)
  • 0x0106_0907_0000_0000 → JSCell header de array (LLInt layout)
  • 0xFFFF_FFFF_FFFF_FFFF → UINT64_MAX, memória não inicializada ou padding
  • 0x7FF8_0000_0000_0000 → NaN canônico do JSC (não é ponteiro)
  • 0x0002_0000_0000_0000 → tag NaN-boxing de double no JSC
  • Valor que muda a cada execução → região com ASLR
  • Valor constante entre execuções → região sem ASLR (útil!)

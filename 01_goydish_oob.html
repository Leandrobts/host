🔴 Bug #1 — JSC DataView Integer Overflow → OOB Write (Sem Gigacage no PS4)

Fonte: oss-sec Dez/2025 · Joseph Goydish II · CVE Pending · Arquivo: JSArrayBufferView.cpp / JSDataView.cpp

📧 oss-sec ADVISORY (seclists.org/oss-sec/2025/q4/292) — 26 Dez 2025 AMBIENTE ORIGINAL: iOS 26.2 · WebKit 8623.1.14.10.9
CRASH: EXC_GUARD (Namespace 31: Gigacage Primitive Partition)
OFFSET: 0xADD7476C — acesso fora do limite Gigacage de 16GB

PoC ORIGINAL (3 linhas de Joseph Goydish):
const buffer = new ArrayBuffer(1024);
const view = new DataView(buffer);
view.setUint32(0xFFFFFFFE, 0x41414141); // 0xFFFFFFFE + 4 = overflow 32-bit → passa bounds check

ARQUIVO VULNERÁVEL: Source/JavaScriptCore/runtime/JSArrayBufferView.cpp
FIX PROPOSTO: __builtin_mul_overflow(index, elementSize, &byteOffset)
STATUS CVE: PENDENTE — Apple notificada simultaneamente ao disclosure público
⚠️ DIFERENÇA CRÍTICA PS4 vs iOS:
iOS/macOS: Gigacage (namespace 31) age como canário — detecta acesso além de 16GB e gera EXC_GUARD. O processo é terminado, mas não há exploitação direta.
PS4 FW 13.04: NÃO implementa Gigacage. O mecanismo não existe no BSFreeBSD do PS4.
→ O overflow passa direto para o hardware sem ser interceptado
→ O write para endereço fora-de-bounds pode corromper memória controlável
→ O que é DoS no iOS pode ser write primitivo no PS4
